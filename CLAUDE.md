# DPmodule — GTport Universal (активный и единственный проект)

> Точка входа для Claude Code. **Источник правды по процессу — `PROJECT_INSTRUCTIONS.md`**
> в этом каталоге; здесь — только выжимка и навигация. Инфраструктура VPS —
> в `../CLAUDE.md`. При конфликте `.md`: `PROJECT_INSTRUCTIONS.md` → `TARGET.md` → остальные.

## Задача (одна, надолго)
Переписать **GTport** (рабочий инструмент диспетчера логистического центра,
учёт дислокации вагонов) из жёстко заточенного под 3 порта (`АЭ`/`УТ-1`/`ГУТ-2`)
в **универсальный** прототип, настраиваемый под любое предприятие: порт-специфичный
хардкод → в БД, парсеры/структуры — расширяемые.

**Способ — greenfield (зафиксировано):** пишем заново на шаблоне `TMPL_backend`
и переносим бизнес-логику из `gtlogic`. Не доработка старого кода по месту.
- `gtlogic` (= старый GTport) — источник **бизнес-логики и правил** (референс, не трогать).
- `TMPL_backend` — источник **архитектуры** (Ports & Adapters, DI в `main.go`).
- `DPmodule` — то, что пишем (seeded из `TMPL_backend`).

## Работа вдвоём (два клона на одном сервере) — воркфлоу
Над проектом работают двое (владелец и тимлид) **под одним логином `alex` на
одном VPS**, но в **разных папках-клонах**, синхронизация только через GitHub
(`git@github.com:Gtport/DPmodule.git`):
```
/home/alex/projects/DPmodule        ← клон владельца
/home/alex/projects/DPmodule-lead   ← клон тимлида
```
Правила (обязательны для людей И для ассистента — соблюдать в любой из папок):
- **Один клон = один автор.** НИКОГДА не редактировать/не запускать git-команды
  в чужой папке. Ассистент работает только в той папке, из которой запущен.
- **В `main` напрямую не коммитим.** Каждая задача — своя ветка
  `git checkout -b feat/<задача>` → `git push -u origin feat/<задача>` → Pull
  Request на GitHub → второй смотрит diff → merge. Так чужая работа не затирается.
- **Перед стартом работы — `git pull`** (или `git fetch && git rebase origin/main`
  в своей ветке), чтобы подтянуть чужие merge. Начинать ветку от свежего `main`.
- **`_reference/seed/` вне git — живёт в каждой папке отдельно.** При обновлении
  справочников синхронизировать вручную (`cp -r`); git их НЕ переносит.
- **Frontend dev-сервер один на всех.** systemd-юнит `dpmodule-frontend`
  (`ng serve` :4200) смотрит на одну папку; второй `ng serve` займёт порт 4200 →
  конфликт. Dev-сервер держит кто-то один, либо второму — другой порт.
- **Конфликт правок** решается через PR-review и `git`, а не копированием файлов
  между папками. Копирование папок мимо git — только для того, что вне git (seed,
  `config.yaml`, `.env`).

## Жёсткие правила (нарушать нельзя без явной просьбы)
- **Поведение GTport — эталон.** Новый код даёт те же результаты на тех же входах.
  **Сначала тест (golden), потом перенос.** Перенос — малыми проверяемыми кусками.
- **Никакого нового хардкода** портов. Новые порт-данные → таблицы + `PortRegistry`,
  не `switch`/`map`. Атрибуты порта — через реестр, не `switch port`.
- **GORM-гибрид (канон):** билдер для CRUD и динамических `UPDATE` (только непустые
  поля); сырой SQL дословно через `gorm.Raw().Scan()`/`gorm.Exec()` для аналитики
  (CTE, `DISTINCT ON`, оконные), DDL, атомарной подмены таблиц. ORM-модель отделена
  от доменной. Снапшот-таблицам (`disl_actual`/`disl_new`) — без `gorm.Model`/хуков.
  Схему ведёт `golang-migrate`, **не** `AutoMigrate`.
- **Время — Московское naive, без часовых поясов** (жёсткий инвариант, `TARGET.md`
  §3.11). Всё время (Excel/JSON/БД/API) — Московское без TZ; отдаём как пришло, **не
  корректируем**. Тип `LocalTime` (без `Z`, `2006-01-02T15:04:05`, `timestamp without
  time zone`). «Сейчас» — только по Москве, из **единого** `clock.Now()` (единственное
  место с `LoadLocation("Europe/Moscow")`); штампы `CreatedAt/UpdatedAt` — через него.
  **Запрещено** в коде: `LoadLocation`/`FixedZone`/`.In`/`.UTC()`/сдвиги `Add(±7h/±10h)`.
  Правило «час ≥ 18 → +1 сутки» (`DateNach`/ID) — бизнес-правило, **не** TZ, остаётся.
- **Silo:** один клиент = один экземпляр = одна БД. `enterprise_id` НЕ вводим.
  RAM-движок дислокации (`actualMap` + индексы + обогащение) переносится **как есть**.
- **Падать громко** на целостности данных. Миграции — идемпотентные, только
  добавляющие. Не переписывать модуль целиком без просьбы — минимальное изменение.
- ⚠️ Не путать **«исторические снимки/бэкапы `disl_actual`»** (упраздняем) с
  **бизнес-историей** `vagon_history`/`vagon_operation` (сохраняем).

## Фронтенд (dev-окружение поднято; наполнение — поздний этап)
Форк шаблона `../TMPL_frontend` (**Angular 21** + ng-zorro + Keycloak) лежит в
`frontend/`. Окружения `src/environments/*`: `moduleId=dpport`,
`keycloak.clientId=iqport-dpport`, `apiBaseUrl=/api`. Стили — только через
`src/styles/tokens.css`. Dev-сервер: systemd user-юнит `dpmodule-frontend`
(`ng serve` на `127.0.0.1:4200`, `--allowed-hosts`, прокси `/api`→`:8080`);
наружу — nginx `app.gtport.ru`→`:4200` с WebSocket-апгрейдом.
`systemctl --user restart|status dpmodule-frontend`.
