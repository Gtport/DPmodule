## Как устроен backend (слои)

Поток данных идёт сверху вниз, зависимости — только «вниз»:

```
handler (HTTP, gin)  →  service (бизнес-логика, RAM-кэши)  →  repository/gorm (БД)
        ↑                          ↑                                   ↑
     port (интерфейсы)      domain (чистые типы)              parser (разбор входных файлов)
```

- **domain** — чистые типы предметной области (вагон, план, справочники), без БД и HTTP.
- **port** — интерфейсы («что умеет хранилище/интеграция»), реализуются в `repository`.
- **service** — вся логика: конвейер обогащения дислокации, приём ЛК, матч плана; держит
  состояние в RAM-кэшах (БД читается на старте, пишется атомарной подменой снимка).
- **repository/gorm** — доступ к PostgreSQL (ORM-модель отделена от доменной).
- **parser** — разбор входных файлов (Excel ЛК, JSON АСУ, Excel план) в доменные типы.
- **handler** — тонкий HTTP-слой (gin), маппинг ошибок в коды; провод в `server.go`.

Точка сборки — `cmd/server/main.go` → `internal/server/server.go` (DI, монтаж маршрутов).

## Frontend (`frontend/`, Angular 21 + ng-zorro + Keycloak)

Standalone-компоненты, состояние на сигналах, стили только через токены. Точка сборки —
`main.ts` → `app.config.ts`. Наружу — nginx `app.gtport.ru` → dev-сервер `:4200` (прокси
`/api` → Go-бэкенд `:8080`).

### Каркас (`src/` и `src/app/`)
| Файл | За что отвечает |
|---|---|
| `src/main.ts` | Точка входа: bootstrap корневого компонента с общей конфигурацией. |
| `src/index.html` | HTML-каркас страницы (язык ru, заголовок, иконка, шрифт); сюда встраивается приложение. |
| `src/styles/tokens.css` | Единый источник дизайн-токенов IQPort (цвета, радиусы, тени, отступы); модули только используют, не переопределяют. |
| `src/app/app.component.ts` | Корневой компонент: оболочка, выводит текущий маршрут (`router-outlet`). |
| `src/app/app.config.ts` | Глобальная настройка: русская локаль, HTTP с авторизацией, регистрация иконок ng-zorro, инициализация авторизации. |
| `src/app/app.routes.ts` | Карта маршрутов: логин, оболочка с разделами; пункты диспетчера из реестра навигации, нереализованные → заглушка (с RBAC). |

### `core/` — общие службы
| Файл | За что отвечает |
|---|---|
| `core/auth/auth.service.ts` | Авторизация: вход логином/паролем через Keycloak, хранение/обновление токена, имя и роли пользователя. |
| `core/auth/auth.guard.ts` | Охрана маршрутов: неавторизованных → `/login`, без нужной роли → `/forbidden`. |
| `core/auth/auth.interceptor.ts` | Подстановка Bearer-токена в запросы к бэкенду модуля (с тихим обновлением токена). |
| `core/api/api-error.ts` | Извлечение человекочитаемого текста ошибки из ответа бэкенда. |
| `core/config/custom-icons.ts` | Самодельные иконки (поезд, судно, склад, тележка), которых нет в стандартном наборе. |
| `core/config/modules.config.ts` | Каталог модулей платформы IQPort (переключатель модулей): название, адрес, роли. |

### `layout/` — общий интерфейс
| Файл | За что отвечает |
|---|---|
| `layout/shell/shell.component.ts` | Каркас интерфейса: верхняя панель, сворачиваемое боковое меню, область контента, подвал. |
| `layout/shell/nav.config.ts` | Реестр разделов бокового меню (путь, подпись, иконка, роли) — единый источник для меню и маршрутов. |
| `layout/module-switcher/module-switcher.component.ts` | Выпадающий переключатель между модулями платформы (только доступные, текущий скрыт). |

### `features/` — рабочие разделы
| Файл | За что отвечает |
|---|---|
| `features/dislocation/dislocation.component.ts` | Экран «Дислокация»: статус-панель системы сверху (переиспользует `plan-status-panel`) + компактный приём ЛК (тулбар «Загрузить/Обновить» → «Обновить дислокацию») + кнопка «Обновить из АСУ» (ручной забор); результат — свёрнутая сводка с «подробнее». Каждый файл ЛК — одной строкой (организация · терминалы · метка/возраст · статус-теги замечаний по своему ОКПО); общие замечания (нет файла/разрыв срезов) — отдельными строками. Кнопка «Обновить дислокацию» и статус — шагом 2 ОТДЕЛЬНОЙ строкой под файлами (визуальное разделение шагов); статус честный (`notReadyReason`: «файлы устарели», «не хватает файлов», «разные срезы»), кнопка заблокирована, если приём не готов. Транзиентный фидбэк (загрузка файла, ошибки, «дислокация обновлена») — всплывающими авто­исчезающими тостами (`NzMessageService`), не в теле. Колонка ~700px. |
| `features/dislocation/dislocation-api.service.ts` | Обмен с бэкендом по ЛК: типы и запросы приёма/контроля/обработки + `asuPull` (ручной забор из АСУ). |
| `features/plan/plan.component.ts` | Экран «План подвода»: загрузка xlsx-плана, кнопка «Пересчитать» (пересчёт выбранной сетки на текущей дислокации без Excel), таблица с портами/«Составом»/«Остаток на 18:00», выбор загрузки из истории, диалог проверки (группы с.ф. + ручные правки индексов ниток с итеративным `revalidate`), статус-панель актуальности сверху. |
| `features/plan/plan-api.service.ts` | Обмен с бэкендом по плану: типы и запросы загрузок, ниток, ячеек портов, с.ф.-кандидатов, проблемных ниток, `prepare`/`recalc`/`revalidate`/`confirm`, статуса актуальности. |
| `features/plan/index-input.component.ts` | Сегментированный ввод индекса поезда 4-3-4 (три поля с зашитыми дефисами): печать слева направо с автопереходом, Backspace возвращает влево, только цифры; выход — собранная строка/событие завершения. |
| `features/plan/plan-status-panel.component.ts` | Статус-панель системы (образец gtport StatusPanel): способ/время обновления дислокации по терминалам (`lk`→ЛК, `asu`/`json`→АСУ) + актуальность планов МА/НК; цвет чипа по возрасту; автообновление раз в минуту. Используется на экранах «План» и «Дислокация». |
| `features/home/home.component.ts` | Стартовая страница модуля (имя и роли пользователя; шаблон-заготовка). |
| `features/placeholder/placeholder.component.ts` | Универсальная заглушка «Раздел в разработке» (заголовок/иконка из маршрута). |

### `pages/` — отдельные страницы
| Файл | За что отвечает |
|---|---|
| `pages/login/login.component.ts` | Страница входа: форма логин/пароль на фирменном анимированном фоне. |
| `pages/forbidden/forbidden.component.ts` | Страница «403 — Доступ запрещён» с кнопкой возврата на главную. |

### Окружения и конфиги проекта
| Файл | За что отвечает |
|---|---|
| `src/environments/*.ts` | Настройки контура (ID модуля, адрес бэкенда, Keycloak); подменяются при сборке. `environment.ts`=база, `.development.ts`=локальная разработка, `.production.ts`=прод, `.uat.ts`=заготовка UAT. |
| `angular.json` | Конфигурация сборки Angular: входные файлы, стили, режимы production/uat/development. |
| `package.json` | Зависимости (Angular 21, ng-zorro, RxJS) и команды (запуск с прокси, сборка, тесты, линт). |
| `proxy.conf.json` | Прокси dev-сервера: `/api` → локальный Go-бэкенд (`localhost:8080`). |
